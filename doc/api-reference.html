    Modules

    Action

    Module for running round actions. Actions encapsulate guessing actions and the data associated with carrying them out. Uses a function builder strategy to easily be able to add new data retrievers, and updaters

    CLI

    Module provides a command line interpreter interface to the Hangman application

    Chunks

    Module to manage word list chunks for a given secret length key

    Chunks.Stream

    Module for generating a chunked stream from a grouped words stream

    Counter

    Module for creating and using letter frequency tallies. Contains a set of functions for tallies, such as adding letters, words, lists, and streams, and retrieving their most common letters

    Dictionary

    Module defines Dictionary common attributes and types. Serves as a central point to access or update such attributes

    Dictionary.Cache

    Module provides access to a dictionary word cache. Provides lookup routines to access chunks, tallys, and random words

    Dictionary.File

    Provides routines to generate various intermediate and cached Dictionary files

    Dictionary.File.Stream

    Module for accessing input file stream for various original and transformed dictionary file types ranging from unsorted, sorted, grouped, chunked

    Game

    Module to create and manage Hangman game abstractions

    Game.Pid.Cache

    Module provides access to a game server pid cache. Pid caching prevents a Game.Server process from having to be created every time. Upon game server startup, the new pid is stored into the pid cache.
    Upon successive game plays with the same game server, the pid does not need to be regenerated, but simply retrieved from the cache

    Game.Server

    Module handles Hangman Game serving to multiple clients.
    Each player's active game state is maintained, until the player process exits

    Guess

    Module implements Guess types

    Hangman.Application

    Main Hangman application

    Pass

    Module defines types Pass.key and Pass.t

    Pass.Cache

    Module provides cache access to the game words pass. Given a player, a game and round number, Pass.Cache maintains a words pass cache of current word passes

    Pass.Writer

    Module is responsible for words pass writes into the Pass.Cache table. The write load is handled through the Pass.Writer.Pool. It distributes write/2 request based on pass key id attribute to workers

    Pass.Writer.Pool

    Module is a Supervisor that supervises a pool of pass writer workers

    Pass.Writer.Worker

    Module handles writer worker functionality used by Pass.Writer. Specifically, Pass.Writer.Worker is a write-operation specific module that performs async writes into Pass.Cache ETS table

    Pattern

    Module handles Hangman pattern updates given guessed letter

    Player

    Module for creating players and maximizing our winning chances in conjunction with the player strategy against the 'implicit' other player the game server. Handles choosing letters, guessing letters and words

    Player.Events

    Module handles player event manager

    Player.Game

    Module handles game playing for synchronous human and robot player types. Handles relationship between Player.FSM, Game.Server and Player.Events

    Player.Logger.Handler

    Module implements event logger handler for Player.Events. Each event is logged to a file named after the player id

    Reduction

    Module implements Reduction key type

    Reduction.Engine

    Module provides access to the words reduction engine

    Reduction.Engine.Pool

    Module is a Supervisor that supervises a pool of word Reduction.Engine workers

    Reduction.Engine.Worker

    Module implements workers that handle Hangman words reduction. Used primarily by Reduction.Engine through reduce_and_store/4 to perform a series of steps

    Reduction.Options

    Module generates Reduction key for use when reducing possible Hangman words set. Used primarily during Round setup

    Round

    Module provides access to game round functions

    Strategy

    Module provides access to a set of functions handling guessing strategy

    Web

    Module provides access to small and simple http web server for playing Hangman games via a tool such as curl or HTTPoison

    Exceptions

    HangmanError

    Module serves as Hangman specific, runtime error module
