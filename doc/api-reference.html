    Modules

    Action

    Module encapsulates Round guess actions and the data associated with carrying them out

    CLI

    Module handles command line interpreter interface to application

    Chunks

    Module to handle word list Chunks for a given length key.
    Internally maintains standardized containers for word lists and keeps track of total word counts and number of containers

    Chunks.Stream

    Module implements transform routine to take a grouped words stream and return a chunked words stream

    Counter

    Module implements a set of functions for working with letter frequency tallies

    Dictionary

    Module defines Dictionary common attributes and types functionality. Serves as a central point to access or update such attributes

    Dictionary.Cache

    Module loads Dictionary.File words into chunks and stores into ETS. Letter frequency tallies are computed and stored into ETS upon startup. Words identified as random are tagged and stored

    Dictionary.File

    Provides abstraction for various Dictionary files

    Dictionary.File.Stream

    Module for accessing input file stream for various Dictionary.File transformed types ranging from unsorted, sorted, grouped, chunked

    Game

    Module defines Hangman Game abstraction which can handle a single Hangman game or multiple Hangman games

    Game.Pid.Cache

    Module implements GenServer functionality to provide Game.Server pid caching. Ultimately relies on the gproc module to administer registry facilities

    Game.Server

    Module implements Hangman Game server using GenServer. Wraps Game abstraction as server state. Runs each game or set of games sequentially

    Guess

    Module implements Guess types

    Hangman.Application

    Main Hangman application

    Pass

    Module defines words Pass.key and Pass.t

    Pass.Cache

    Given a player, a game and round number, Pass.Cache maintains a words pass cache

    Pass.Writer

    Module implements words pass write functionality into the Pass.Cache ETS table

    Pass.Writer.Pool

    Module is a Supervisor that supervises a pool of pass writer workers

    Pass.Writer.Worker

    Module is a GenServer that implements writer worker functionality. Specifically, Pass.Writer.Worker is a write-operation specific module that performs async writes into Pass.Cache ETS table

    Pattern

    Module handles Hangman pattern updates given guessed letter

    Player

    Module handles Player abstraction and defines Player type. Implements player functionality to choose letters, guess letters and words. Heavily relies upon Round and Action

    Player.Events

    Module implements GenEvent manager for use with player

    Player.Game

    Module handles game playing for synchronous human and robot player types. Handles relationship between Player.FSM, Game.Server and Player.Events

    Player.Logger.Handler

    Module implements event logger handler for Player.Events

    Reduction

    Module implements Reduction key type

    Reduction.Engine

    Module implements words reduction engine

    Reduction.Engine.Pool

    Module is a Supervisor that supervises a pool of word Reduction.Engine workers

    Reduction.Engine.Worker

    Module is a worker GenServer that implements Hangman words reduction, primarily through reduce_and_store/4

    Reduction.Options

    Module generates Reduction key for use when reducing possible Hangman words set. Used primarily during Round setup

    Round

    Module to implement Hangman game round abstraction

    Strategy

    Handles letter guessing strategy for Player

    Web

    Implements Hangman http web server for playing Hangman games

    Exceptions

    HangmanError

    Module serves as Hangman specific, runtime error module
