    Modules

    Hangman.Action

    Defines the Action type used in Player.Action

    Hangman.Action.Human

    Implements human action player specific functionality

    Hangman.Action.Robot

    Implements robot action player specific functionality

    Hangman.Application

    Main Hangman application

    Hangman.CLI

    Module provides a command line interpreter interface to the Hangman application

    Hangman.CLI.Handler

    Module drives Player.Controller, while setting up the proper Game server and Event consumer states beforehand

    Hangman.Chunks

    Module to manage word list chunks for a given secret length key

    Hangman.Chunks.Stream

    Module for generating a chunked stream from a grouped words stream

    Hangman.Counter

    Module for creating and using letter frequency tallies. Contains a set of functions for tallies, such as adding letters, words, lists, and streams, and retrieving their most common letters

    Hangman.Dictionary

    Module defines Dictionary common attributes and types. Serves as a central point to access or update such attributes

    Hangman.Dictionary.Cache

    Module provides access to a dictionary word cache. Provides lookup routines to access chunks, tallys, and random words

    Hangman.Dictionary.File.ChunkedReader
    Hangman.Dictionary.File.Chunker
    Hangman.Dictionary.File.GroupedReader
    Hangman.Dictionary.File.Grouper
    Hangman.Dictionary.File.OriginalReader
    Hangman.Dictionary.File.Reader

    Module for accessing input file stream for various original and transformed dictionary file types ranging from original, sorted, grouped, chunked

    Hangman.Dictionary.File.SortedReader
    Hangman.Dictionary.File.Sorter
    Hangman.Dictionary.File.Transformer

    This module specifies the interface routines to transform the input dictionary file to a new file, based on the logic of the transform function

    Hangman.Dictionary.File.Writer
    Hangman.Dictionary.Transformer

    The original Dictionary file is transformed into intermediary representations. Given an original dictionary file f1, this file may be transformed a few times until it is suitable to be loaded into ETS. E.g. f1 -> f2 -> f3 -> f4

    Hangman.Game

    Module to create and manage Hangman game abstractions

    Hangman.Game.Event.Manager
    Hangman.Game.Pid.Cache

    Module provides access to a game server pid cache. Pid caching prevents a Game.Server process from having to be created every time. Upon game server startup, the new pid is stored into the pid cache.
    Upon successive game plays with the same game server, the pid does not need to be regenerated, but simply retrieved from the cache

    Hangman.Game.Server

    Module handles Hangman Game serving to multiple clients.
    Each player's active game state is maintained, until the player process exits

    Hangman.Game.Server.Stub
    Hangman.Guess

    Module implements Guess types

    Hangman.Letter.Strategy

    Module provides access to a set of functions handling letter guessing strategy

    Hangman.Pass

    Module defines types Pass.key and Pass.t

    Hangman.Pass.Cache

    Module provides cache access to the game words pass. Given a player, a game and round number, Pass.Cache maintains a words pass cache of current word passes. The words are represented by Chunks

    Hangman.Pass.Cache.Writer

    Module is responsible for synchronous chunk pass writes into the Pass.Cache table

    Hangman.Pattern

    Module handles Hangman pattern updates given guessed letter

    Hangman.Player
    Hangman.Player.Alert.Handler

    Module implements alert event handler for Hangman.Events.Manager. Each event is displayed to the user as an alert feed event

    Hangman.Player.Controller

    Serves as External API into Player functionality.
    Serves to manage players

    Hangman.Player.FSM

    Module implements a non-process player fsm which handles managing the state of types implemented through the Player Action protocol

    Hangman.Player.Generic

    Provides Generic player routine implementations used by the Player.Action protocol. Fills in the Player gaps before the specific Action implementation kicks in

    Hangman.Player.Logger.Handler

    Module implements event logger handler for Hangman.Events.Manager. Each event is logged to a file named after the player id

    Hangman.Player.Types
    Hangman.Player.Worker

    GenServer module to implement Player Worker, managed by Player.Controller. Module uses gproc to manage process registry of workers

    Hangman.Reduction

    Module implements Reduction key type

    Hangman.Reduction.Engine

    Module provides access to the words reduction engine

    Hangman.Reduction.Engine.Pool

    Module is a Supervisor that supervises a pool of word Reduction.Engine workers

    Hangman.Reduction.Engine.Worker

    Module implements workers that handle Hangman words reduction. Used primarily by Reduction.Engine through reduce_and_store/4 to perform a series of steps

    Hangman.Reduction.Options

    Module generates Reduction key for use when reducing possible Hangman words set. Used primarily during Round setup

    Hangman.Round

    Module provides access to game round functions

    Hangman.Simple.Registry

    Module implements a simple stash registry keeping track of active pids, active ids, and their values e.g. Player.t or Game.t

    Hangman.Web

    Module provides access to a http web server for playing Hangman games via a tool such as curl or HTTPoison

    Hangman.Web.Flow

    Module distributes the game requests to the player workers and then collects the resultant game information and combines it in the proper order

    Hangman.Web.Shard.Handler

    Module runs game play for the given web shard of secrets as determined by Web.Flow. Basically, runs a chunk of the overall original secrets vector

    Exceptions

    HangmanError

    Module serves as Hangman specific, runtime error module

    Protocols

    Hangman.Player.Action

    Module implements player action functionality for various player types via protocol mechanism
