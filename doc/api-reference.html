    Modules

    Hangman.Action.Human

    Implements human action player specific functionality

    Hangman.Action.Robot

    Implements robot action player specific functionality

    Hangman.Application

    Main Hangman application

    Hangman.CLI

    Module provides a command line interpreter interface to the Hangman application

    Hangman.CLI.Handler

    Module drives Player.Controller, while setting up the proper Game server and Event consumer states beforehand

    Hangman.Counter

    Module for creating and using letter frequency tallies. Contains a set of functions for tallies, such as adding letters, words, lists, and streams, and retrieving their most common letters

    Hangman.Dictionary

    Module provides central point to perform dictionary functions for clients for types, paths, random words, and lookup functionality

    Hangman.Dictionary.Cache

    Module implements a GenServer process providing access to a dictionary word cache. Handles lookup routines to access words, tallys, and random words

    Hangman.Dictionary.ETS

    Module provides access to a dictionary word cache powered by ETS. Provides lookup routines to access words, tallys, and random words. Handles all ETS specific operations while Dictionary serves as a wrapper

    Hangman.Dictionary.Ingestion

    Module handles the ingestion of hangman dictionary words through the coordination of Ingestion.First.Flow and Ingestion.Cache.Flow

    Hangman.Game

    Module to create and manage Hangman game abstractions

    Hangman.Game.Event.Manager

    Module implements GenStage event manager which handles event dispatching given consumer backpressure

    Hangman.Game.Server

    Module handles Hangman Game serving to multiple clients.
    Each player's active game state is maintained, until the player process exits

    Hangman.Game.Server.Controller

    Module provides dynamic startup of Game Servers

    Hangman.Game.Server.Stub
    Hangman.Guess

    Module implements Guess types

    Hangman.Ingestion.Cache.Flow

    Loads partitioned dictionary word files into ingestion table piecewise through words list chunks. Random words are generated from these words lists

    Hangman.Ingestion.First.Flow

    Module implements the introductory ingestion flow logic given an initial hangman dictionary file. Serves to split dictionary into windowed word list chunks storing them in intermediate cached files arranged by word key length

    Hangman.Letter.Retrieval.Common
    Hangman.Letter.Retrieval.Strategy
    Hangman.Letter.Strategy

    Module provides access to a set of functions handling letter guessing strategy

    Hangman.Pass

    Module defines types Pass.key and Pass.t

    Hangman.Pass.Cache

    Module provides cache access to the game words pass. Given a player, a game and round number, Pass.Cache maintains a words pass cache of current word passes. The words are represented by Words.t

    Hangman.Pass.Cache.Writer

    Module is responsible for synchronous words pass writes into the Pass.Cache table

    Hangman.Pass.Reduction

    Module provides pass reduction interface for reduction engine workers

    Hangman.Pattern

    Module handles Hangman pattern updates given guessed letter

    Hangman.Player

    Defines Player id and key

    Hangman.Player.Alert.Handler

    Module implements alert event handler for Hangman.Events.Manager. Each event is displayed to the user as an alert feed event

    Hangman.Player.Controller

    Serves as External API into Player functionality.
    Serves to manage players

    Hangman.Player.FSM

    Module implements a non-process player fsm which handles managing the state of types implemented through Player and the Player Action protocol

    Hangman.Player.Logger.Handler

    Module implements event logger handler for Hangman.Events.Manager. Each event is logged to a file named after the player id

    Hangman.Player.Types

    Defines Player types and mapping

    Hangman.Player.Worker

    GenServer module to implement Player Worker, managed by Player.Controller. Module uses gproc to manage process registry of workers

    Hangman.Reduction

    Module implements Reduction key type

    Hangman.Reduction.Engine

    Module provides access to the words reduction engine

    Hangman.Reduction.Engine.Pool

    Module is a Supervisor that supervises a pool of word Reduction.Engine workers

    Hangman.Reduction.Engine.Worker

    Module implements workers that handle Hangman words reduction. Used primarily by Reduction.Engine through reduce_and_store/4 to perform a series of steps

    Hangman.Reduction.Options

    Module generates Reduction key for use when reducing possible Hangman words set. Used primarily during Round setup

    Hangman.Round

    Module provides access to game round functions

    Hangman.Shard.Flow

    Module splits the game args into shards which is distributed to shard handlers. The resultant game information is collected and combined in the proper order

    Hangman.Shard.Handler

    Module runs game play for the given shard of secrets as determined by Shard.Flow. Basically, runs a chunk of the overall original secrets vector

    Hangman.Simple.Registry

    Module implements a simple stash registry keeping track of active pids, active ids, and their values e.g. Player.t or Game.t

    Hangman.Web

    Module provides access to a http web server for playing Hangman games via a tool such as curl or HTTPoison

    Hangman.Words

    Module to manage word lists for a given secret length key

    Exceptions

    HangmanError

    Module serves as Hangman specific, runtime error module

    Hangman.Player.Action
