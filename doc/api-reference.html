    Modules

    Action

    Module encapsulates hangman round guess actions and the data associated with carrying them out

    CLI

    Module handles command line interpreter interface to application

    Chunks

    Module to handle word list chunks for a given length key. Internally maintains standardized containers for word lists and keeps track of total word and chunks counts

    Chunks.Stream

    Module implements transform routine to take a grouped words stream and return a chunked words stream

    Counter

    Module implements a set of functions for working with letter frequency counters

    Dictionary

    Module defines dictionary common attributes and types functionality. Serves as a central point to access or update such attributes

    Dictionary.Cache

    Module loads dictionary file words into chunks and stores into ETS. Letter frequency tallies are computed and stored into ETS upon startup. Words identified as ‘random’ are tagged and stored

    Dictionary.File

    Provides abstraction for various dictionary files

    Dictionary.File.Stream

    Module for accessing input file stream for various dictionary file transformed types range from unsorted (original), sorted, grouped, chunked (last)

    Game

    Module defines game abstraction which can handle a single hangman game or multiple hangman games

    Game.Pid.Cache

    Module provides game server pid caching so a game server doesn’t need to be reloaded on every use

    Game.Server

    Module implements hangman game using GenServer process. Wraps hangman game abstraction as server state

    Guess

    Module primarily implements guess types

    Pass

    Module implements words pass types and words pass module struct

    Pass.Cache

    Module implements ets table owning process for hangman words pass state for a given player, round number

    Pass.Cache.Stub

    Stub module to mimic Pass Cache functionality

    Pass.Writer

    Module implements words pass writer functionality. Write load is handled through pool. Distributes write request based on pass key id attribute (name) Pool size writer workers are started up as part of writer pool Pool supervisor supervises writer workers

    Pass.Writer.Pool

    Module is a Supervisor that supervises a pool of pass writer workers

    Pass.Writer.Worker

    Module is a GenServer that implements writer worker functionality. Specifically, module is a write operation specific module that performs async write into pass ets table

    Pattern

    Module handles hangman pattern update given letter

    Player

    Module handles player abstraction and defines player type

    Player.Async.Echo

    Module serves as an asynchronous echo server for the asynchronous robot fsm player type

    Player.Events

    Module implements events manager for player abstraction

    Player.FSM

    Module implements state machine model for player

    Player.Game

    Module handles relationship between player, game server and player event server

    Player.Logger.Handler

    Module implements event logger handler for player events Each event is logged to a file named after player id (name)

    Reduction

    Module implements reduction abstraction key type only

    Reduction.Engine

    Module implements words reduction engine

    Reduction.Engine.Pool

    Module is a Supervisor that supervises a pool of word reduction workers

    Reduction.Engine.Worker

    Module is a GenServer that implements worker word reducer functionality

    Round

    Module to implement hangman game round abstraction

    Strategy

    Handles letter guessing strategy for player

    Strategy.Options

    Module generates reduction key for use when reducing possible words set

    Exceptions

    HangmanError

    Module serves as hangman specific, runtime error module
