    Modules

    Hangman
    Hangman.Application

    Main application callback module

    Hangman.CLI

    Hangman.CLI Module handles command line interpreter interface to running Hangman games

    Hangman.Chunks

    Module to handle Hangman word list chunks for a given length

    Hangman.Chunks.Stream

    Module implements transform routine to take a grouped words stream and return a chunked words stream

    Hangman.Counter

    Hangman.Counter A set of functions for working with letter frequency counters Counters are key value stores where keys are letter strings and values are positive integers

    Hangman.Dictionary.Attribute.Tokens

    Module defines common attribute ‘tokens’ used by dictionary functionality. Serves as a central point to access or update such tokens

    Hangman.Dictionary.Cache.Server
    Hangman.Dictionary.Cache.Timing.Profile

    Module to time dictionary cache server

    Hangman.Dictionary.File

    Provides abstraction for various dictionary files

    Hangman.Dictionary.File.Stream

    Module for accessing input file stream for various dictionary file types

    Hangman.Game

    Module defines game abstraction which can handle a single hangman game or multiple hangman games

    Hangman.Game.Pid.Cache.Server

    Module provides game server pid caching so a game server doesn’t need to be reloaded on every use

    Hangman.Game.Server

    Module implements hangman game using GenServer process. Wraps hangman game abstraction as server state

    Hangman.Game.Server.Supervisor

    Module implements supervisor behaviour

    Hangman.Game.System.Supervisor

    Module implements supervisor behaviour

    Hangman.Guess

    Module primarily implements guess types

    Hangman.Guess.Action

    Module encapsulates hangman round actions and the data associated with carrying them out

    Hangman.Pass

    Module implements pass abstraction types and pass module struct

    Hangman.Pass.Server

    Module implements ets table owning process for hangman words pass state for a given player, round number

    Hangman.Pass.Server.Stub

    Stub module to mimic Pass Server functionality

    Hangman.Pass.Writer

    Module implements words pass writer functionality. Write load is handled through pool. Distributes write request based on pass key id attribute (name) Pool size writer workers are started up as part of writer pool Pool supervisor supervises writer workers

    Hangman.Pass.Writer.Pool.Supervisor

    Module is a Supervisor that supervises a pool of pass writer workers

    Hangman.Pass.Writer.Worker

    Module is a GenServer that implements writer worker functionality. Specifically, module is a write operation specific module that performs async write into pass ets table

    Hangman.Pattern

    Module handles hangman pattern update given letter

    Hangman.Player

    Module handles player abstraction and defines player type

    Hangman.Player.Async.Echo

    Module serves as an asynchronous echo server for the asynchronous robot fsm player type

    Hangman.Player.Events.Server

    Module implements events manager for player abstraction

    Hangman.Player.Events.Supervisor

    Module implements supervisor functionality overseeing dynamically started player events servers

    Hangman.Player.FSM

    Module implements state machine model for player

    Hangman.Player.Game

    Module handles relationship between player, game server and player event server

    Hangman.Player.Group.Supervisor

    Module is a second line supervisor as it supervises two first-line supervisors

    Hangman.Player.Logger.Handler

    Module implements event logger handler for player events Each event is logged to a file named after player id (name)

    Hangman.Player.Round

    Module to implement player game round abstraction

    Hangman.Player.Supervisor

    Module implements supervisor functionality, overseeing dynamically started player fsms

    Hangman.Player.System.Supervisor

    Module supervises all components necessary for player and strategic player game play including dictionary cache server, word pass server, reduction engine, pass writer, and player events and fsm supervisor

    Hangman.Reduction

    Module implements reduction abstraction key type only

    Hangman.Reduction.Engine

    Module implements words reduction engine

    Hangman.Reduction.Engine.Pool.Supervisor

    Module is a Supervisor that supervises a pool of word reduction workers

    Hangman.Reduction.Engine.Worker

    Module is a GenServer that implements worker word reducer functionality

    Hangman.Strategy

    Handles letter guessing strategy for player

    Hangman.Strategy.Options

    Module generates reduction key for use when reducing possible words set

    Hangman.Supervisor

    Module is the root level supervisor

    Exceptions

    Hangman.Error

    Module serves as hangman specific, runtime error module
